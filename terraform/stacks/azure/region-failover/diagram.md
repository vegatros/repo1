## Failover Flow

```
  Normal Operation
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  User вҶ’ Traffic Manager вҶ’ Primary VM (eastus2) вҶ’ SQL Failover R/W endpoint
                                                 вҶ’ Storage primary endpoint

  Primary Region Failure
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  1. Traffic Manager probe fails 3x on primary     (30 seconds)
  2. DNS re-routes traffic to secondary VM          (+ 60s TTL)
  3. SQL Failover Group promotes secondary          (up to 60 min grace)
  4. Storage GZRS: read from secondary endpoint     (automatic)

  Recovery (Failback)
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  1. Primary region recovers
  2. SQL Failover Group auto-reseeds to original primary
  3. Manual or automatic failback via Traffic Manager priority
```
